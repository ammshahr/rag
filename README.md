# گزارش فنی پیاده‌سازی سیستم RAG پایه روی اخبار فارسی

# 

# این پروژه با هدف ساخت یک سامانه بازیابی و پاسخ‌دهی (RAG) روی آرشیو اخبار خبرگزاری فارس در سه مرحله اصلی اجرا شد. ۱. آماده‌سازی داده‌ها: ابتدا داده‌های خام پاک‌سازی و نرمال‌سازی شدند (یکسان‌سازی کاراکترهای "ک" و "ی" و حذف فواصل اضافی). سپس فیلترینگ بر اساس طول متن انجام و یک زیرمجموعه ۱۰,۰۰۰ تایی برای پردازش انتخاب شد. ۲. ساخت مدل RAG: متن‌ها به قطعات (Chunks) ۸۰۰ کاراکتری با هم‌پوشانی ۱۵۰ کاراکتر تقسیم شدند. برای افزایش دقت بازیابی، از رویکرد ترکیبی (Hybrid) استفاده شد؛ به این صورت که بردارهای معنایی حاصل از مدل MiniLM-L12-v2 با بردارهای لغوی TF-IDF ترکیب شدند. جهت جستجوی سریع برداری نیز از کتابخانه Faiss استفاده گردید. ۳. ارزیابی: سیستم نهایی با ۱۵ پرسش آزمایشی در حوزه‌های ورزشی، سیاسی و اقتصادی تست شد. مکانیزم پاسخ‌دهی فعلی، ۳ سند برتر بازیابی شده را به عنوان پاسخ نمایش می‌دهد. نتیجه‌گیری: سیستم در بخش بازیابی (Retrieval) و یافتن مستندات مرتبط سرعت و دقت قابل قبولی دارد، اما به دلیل عدم وجود یک LLM مولد (Generative)، پاسخ‌ها صرفاً الحاق مستندات هستند. برای ارتقا پیشنهاد می‌شود از مدل‌های Embedding اختصاصی فارسی و یک مدل زبانی بزرگ برای تولید متن نهایی استفاده شود.
